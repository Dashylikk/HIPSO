## 🎯 Призначення

`HIPSO` розв'язує **задачу глобальної оптимізації** мультимодальної цільової функції у *r*-вимірному дійсному просторі:

```math
\min_{\mathbf{x} \in [\text{lo}, \text{hi}]^r} \; f(\mathbf{x})
```

Метод призначений для задач, де класичні алгоритми (рій часток, генетичний, імітація відпалу) застрягають у локальних мінімумах через велику кількість конкуруючих оптимумів. Типові області застосування:

- Оптимізація рецептур композиційних матеріалів
- Проєктування схем армування шаруватих пакетів
- Налаштування параметрів нейронних мереж
- Інженерна оптимізація з обмеженнями міцності, маси, вартості

---

## ⚙️ Особливості методу

HIPSO поєднує **9 механізмів** дослідження простору рішень:

| № | Механізм | Призначення |
|---|---|---|
| 1 | Ройовий пошук (PSO-ядро) | Базова динаміка руху агентів до особистого та групового кращого |
| 2 | Розшарування на *M* груп | Паралельне дослідження різних областей простору |
| 3 | Внутрішньогрупове імунне стиснення | Усунення дублювання — близькі агенти видаляються |
| 4 | Міжгрупове змагання | Слабкіші групи розформовуються |
| 5 | Адаптивне зменшення *M* | З часом рій консолідується в найкращій області |
| 6 | Когерентність груп | Агенти, що «забрели» в чужу зону, повертаються до своєї |
| 7 | Відбиття від меж простору | Обмеження області пошуку з редукцією швидкості |
| 8 | Випадкова рівномірна ініціалізація | Незміщений початковий пошук по всьому простору |
| 9 | Рання зупинка за стагнацією | Виходимо, якщо протягом 150 ітерацій немає поліпшення |

## 📐 Математична основа

### Ройова динаміка (PSO-ядро)

Швидкість *i*-го агента у *j*-му вимірі оновлюється за формулою:

```math
v_{ij}(t+1) = a \cdot v_{ij}(t) + b \cdot u \cdot (p^{best}_{ij} - x_{ij}(t)) + c \cdot u \cdot (g^{best}_{j} - x_{ij}(t))
```

```math
x_{ij}(t+1) = x_{ij}(t) + v_{ij}(t+1)
```

де:
- $a$ — коефіцієнт інерції
- $b$ — когнітивний коефіцієнт («ностальгія» — тяжіння до власного кращого)
- $c$ — соціальний коефіцієнт («ройова мораль» — тяжіння до кращого в групі)
- $u \in [0, 1]$ — **спільне** випадкове число для обох членів (особливість HIPSO)
- $p^{best}_i$ — найкраща позиція *i*-го агента за всю історію
- $g^{best}_g$ — найкраща позиція в групі *g*

### Когерентність груп

Якщо агент опинився ближче до центроїда *чужої* групи, ніж власної, він примусово повертається:

```math
v_{ij}(t+1) = a \cdot (\bar{x}_{gj} - x_{ij}(t))
```

де $\bar{x}_g$ — центроїд групи *g*.

### Відбиття від меж простору

```math
\text{якщо } x_{ij} < \text{lo} \text{ або } x_{ij} > \text{hi}: \quad v_{ij} \leftarrow -\frac{v_{ij}}{2}, \quad x_{ij} \leftarrow \text{clamp}(x_{ij})
```

### Внутрішньогрупове імунне стиснення

Активується кожні $z$ ітерацій. Радіус стиснення:

```math
R_{in}(t) = \frac{\sqrt{r}}{N_m \cdot M \cdot 2 \cdot (1+t)}
```

де $N_m$ — кількість агентів у групі, $r$ — розмірність задачі.

Якщо два агенти ближче ніж $R_{in}$, гірший із них **видаляється і генерується наново** в околиці лідера групи з радіусом:

```math
\sigma = R_{in} \cdot \left( \frac{p_{min}}{1 - p_{min}} \cdot (N_m - 1) \right)^{1/r}
```

де $p_{min} = 0.05$ — імовірність випадкового збігу.

### Міжгрупове змагання

Активується кожні $f$ ітерацій. У кожному циклі змагання:

- **Переможець** (найкраща група) скидає лічильник поразок: $\text{losses}_{winner} = 0$
- **Програш** (найгірша група) накопичує поразку: $\text{losses}_{loser} += 1$
- Якщо $\text{losses}_{loser} \geq C_{max}$ — група розформовується, її агенти перерозподіляються до найближчих груп за відстанню

```math
\text{group}_i \leftarrow \arg\min_{g \in \text{keep}} \|\mathbf{x}_i - \bar{x}_g\|, \quad \forall i \in \text{loser}
```

### Метрика якості параметрів

Для дослідження оптимальних гіперпараметрів використовується метрика Зіборова:

```math
F_{param} = |f_{best}| \cdot \text{Iter}_{best}
```

— чим менше, тим краще (швидкий і точний пошук).

## 🚀 Швидкий старт

### Базовий приклад

```python
from HIPSO import HIPSO, rastrigin

# Створення оптимізатора
optimizer = HIPSO(
    func=rastrigin,        # цільова функція
    dim=8,                 # розмірність задачі
    bounds=(-5.12, 5.12),  # межі простору пошуку
    
    # Параметри ройової динаміки
    a=0.77,    # інерція
    b=1.53,    # ностальгія (b ≈ 2a)
    c=1.64,    # ройова мораль (c ≈ 2a)
    
    # Параметри імунної системи
    nPop=80,   # розмір популяції
    M=5,       # початкова кількість груп
    z=11,      # період внутрішньогрупового стиснення (просте число)
    f=13,      # період міжгрупового змагання (просте число, f ≠ z)
    cmax=5,    # максимум поразок групи перед розформуванням
    
    max_iter=500,
)

# Запуск оптимізації
best_f, iter_used = optimizer.run()
print(f"Найкраще значення: {best_f:.6f} за {iter_used} ітерацій")
```

### Власна цільова функція

```python
def my_function(x):
    """Будь-яка функція f: R^n -> R"""
    return sum(xi**2 for xi in x) - 5*math.cos(x[0])

optimizer = HIPSO(func=my_function, dim=10, bounds=(-10, 10))
best_f, _ = optimizer.run()
```

## 🎯 Ключові результати

### Закономірність $b \approx c \approx 2a$

Найважливіше відкриття дослідження — параметри ройового пошуку завжди задовольняють співвідношення:

```math
b \approx c \approx 2a, \quad \text{з відхиленням не більше 15 \%}
```

**Перевірка по результатах:**

| r | $b/a$ | $c/a$ |
|---:|---:|---:|
| 2  | 2.16 | 2.28 |
| 8  | 2.05 | 2.26 |
| 20 | 1.90 | 2.00 |

Це **узгоджується з теоремою стабільності Trelea (2003)** та означає, що:

> На практиці достатньо обрати лише один параметр $a \in [0.65; 0.85]$, а решту автоматично обчислити як $b = c = 2a$.

### Інші висновки

1. **Початкова кількість груп** $M = 5$ оптимальна для всіх досліджених розмірностей
2. **Розмір популяції** $nPop$ зростає лінійно з $r$ (від 46 для $r=2$ до 99 для $r=20$)
3. **Періоди $f, z$** оптимальні в діапазоні простих чисел [11, 23]; обов'язково $f \neq z$
4. **$C_{max}$** слабо чутливий до розмірності — стабільно ≈ 5

## ⚙️ Параметри

### Параметри конструктора `HIPSO()`

| Параметр | Тип | За замовч. | Опис |
|---|---|---|---|
| `func` | `Callable` | — | Цільова функція $f: \mathbb{R}^r \to \mathbb{R}$ |
| `dim` | `int` | — | Розмірність задачі $r$ |
| `bounds` | `(float, float)` | — | Межі простору пошуку $[\text{lo}, \text{hi}]$ |
| `a` | `float` | `0.75` | Інерція (рекомендовано 0.65–0.85) |
| `b` | `float` | `1.5` | Ностальгія (рекомендовано $\approx 2a$) |
| `c` | `float` | `1.5` | Ройова мораль (рекомендовано $\approx 2a$) |
| `nPop` | `int` | `50` | Розмір популяції |
| `M` | `int` | `4` | Початкова кількість груп |
| `z` | `int` | `11` | Період внутрішньогрупового стиснення |
| `f` | `int` | `13` | Період міжгрупового змагання |
| `cmax` | `int` | `4` | Максимум поразок групи |
| `max_iter` | `int` | `1000` | Максимум ітерацій |
| `eps` | `float` | `1e-10` | Поріг зупинки при досягненні |
| `p_min` | `float` | `0.05` | Імовірність випадкового збігу |

### Метод `.run()`

Повертає кортеж:
- `best_f` (float) — найкраще знайдене значення цільової функції
- `iter_used` (int) — кількість ітерацій, на якій знайдено `best_f`
