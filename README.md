# HIPSO — Hybrid Immune Particle Swarm Optimization

> Гібридний імунно-ройовий метод глобальної оптимізації мультимодальних функцій у дійсному просторі змінних.



##  Призначення

`HIPSO` розв'язує **задачу глобальної оптимізації** мультимодальної цільової функції у *r*-вимірному дійсному просторі:

```math
\min_{\mathbf{x} \in [\text{lo}, \text{hi}]^r} \; f(\mathbf{x})
```

Метод призначений для задач, де класичні алгоритми (рій часток, генетичний, імітація відпалу) застрягають у локальних мінімумах через велику кількість конкуруючих оптимумів. Типові області застосування:

- Оптимізація рецептур композиційних матеріалів
- Проєктування схем армування шаруватих пакетів
- Налаштування параметрів нейронних мереж
- Інженерна оптимізація з обмеженнями міцності, маси, вартості

---

##  Особливості методу

HIPSO поєднує **9 механізмів** дослідження простору рішень:

| № | Механізм | Призначення |
|---|---|---|
| 1 | Ройовий пошук (PSO-ядро) | Базова динаміка руху агентів до особистого та групового кращого |
| 2 | Розшарування на *M* груп | Паралельне дослідження різних областей простору |
| 3 | Внутрішньогрупове імунне стиснення | Усунення дублювання — близькі агенти видаляються |
| 4 | Міжгрупове змагання | Слабкіші групи розформовуються |
| 5 | Адаптивне зменшення *M* | З часом рій консолідується в найкращій області |
| 6 | Когерентність груп | Агенти, що «забрели» в чужу зону, повертаються до своєї |
| 7 | Відбиття від меж простору | Обмеження області пошуку з редукцією швидкості |
| 8 | Випадкова рівномірна ініціалізація | Незміщений початковий пошук по всьому простору |
| 9 | Рання зупинка за стагнацією | Виходимо, якщо протягом 150 ітерацій немає поліпшення |

---

##  Математична основа: 9 кроків алгоритму

Розглянемо задачу мінімізації цільової функції $f(\mathbf{x})$ у *r*-вимірному просторі $\mathbf{x} \in [\text{lo}, \text{hi}]^r$. Метод HIPSO виконується як послідовність дев'яти кроків: перші два — ініціалізація, кроки 3-9 повторюються в основному циклі до досягнення критерію зупинки.

### Крок 1. Ініціалізація популяції

Згенерувати $nPop$ агентів зі **випадковими рівномірно розподіленими позиціями** по всьому простору пошуку та невеликими випадковими швидкостями:

```math
x_{ij}(0) \sim U(\text{lo}, \text{hi}), \quad i=1,\dots,nPop, \quad j=1,\dots,r
```

```math
v_{ij}(0) \sim U(-0.1 \cdot (\text{hi}-\text{lo}),\; 0.1 \cdot (\text{hi}-\text{lo}))
```

Розподілити агентів по $M$ групах за принципом round-robin:

```math
g_i = i \mod M, \quad i=0,1,\dots,nPop-1
```

Лічильники поразок груп обнуляються: $\text{losses}_g = 0, \; g=1,\dots,M$.

### Крок 2. Оцінка початкового фітнесу та визначення кращих рішень

Обчислити $f(\mathbf{x}_i)$ для всіх агентів. Встановити **особисте найкраще** ($p^{best}$) як поточні позиції та **групове найкраще** ($g^{best}$) як кращі агенти в кожній групі:

```math
\mathbf{p}^{best}_i = \mathbf{x}_i(0), \quad f^{p}_i = f(\mathbf{x}_i)
```

```math
\mathbf{g}^{best}_g = \arg\min_{i:\,g_i = g} f^{p}_i, \quad g=1,\dots,M
```

```math
f^{best} = \min_{i} f^{p}_i
```

### Крок 3. Ройова динаміка (PSO-ядро)

Для кожного агента *i* у групі $g_i$ оновити швидкість і позицію. Особливість HIPSO — **спільне** випадкове число $u$ для когнітивної та соціальної складових:

```math
u \sim U(0, 1)
```

```math
v_{ij}(t+1) = a \cdot v_{ij}(t) + b \cdot u \cdot (p^{best}_{ij} - x_{ij}(t)) + c \cdot u \cdot (g^{best}_{g_i,j} - x_{ij}(t))
```

```math
x_{ij}(t+1) = x_{ij}(t) + v_{ij}(t+1)
```

де $a$ — інерція, $b$ — ностальгія (тяжіння до власного кращого), $c$ — ройова мораль (тяжіння до кращого в групі). Оптимально $b \approx c \approx 2a$.

### Крок 4. Відбиття від меж простору пошуку

Якщо агент вийшов за межі допустимої області, відбити його з **редукцією швидкості вдвічі**:

```math
\text{якщо } x_{ij} < \text{lo}: \quad x_{ij} \leftarrow \text{lo}, \quad v_{ij} \leftarrow -\frac{v_{ij}}{2}
```

```math
\text{якщо } x_{ij} > \text{hi}: \quad x_{ij} \leftarrow \text{hi}, \quad v_{ij} \leftarrow -\frac{v_{ij}}{2}
```

Це запобігає необмеженому зростанню швидкості та зберігає енергію руху у напрямку, протилежному межі.

### Крок 5. Когерентність груп

Обчислити центроїди всіх *M* груп:

```math
\bar{\mathbf{x}}_g = \frac{1}{|G_g|} \sum_{i \in G_g} \mathbf{x}_i, \quad G_g = \{i : g_i = g\}
```

Якщо агент опинився ближче до центроїда **чужої** групи, ніж до своєї, його швидкість примусово перенаправляється до власної групи:

```math
\text{якщо } \exists\, k \neq g_i:\; \|\mathbf{x}_i - \bar{\mathbf{x}}_k\| < \|\mathbf{x}_i - \bar{\mathbf{x}}_{g_i}\|, \quad \text{тоді:}
```

```math
v_{ij}(t+1) \leftarrow a \cdot (\bar{x}_{g_i,j} - x_{ij}(t))
```

Цей механізм запобігає «розпорошуванню» груп та зберігає паралельне дослідження різних областей.

### Крок 6. Оновлення особистого та групового кращого

Перерахувати фітнес у нових позиціях і оновити $p^{best}, g^{best}, f^{best}$:

```math
\text{якщо } f(\mathbf{x}_i(t+1)) < f^{p}_i: \quad \mathbf{p}^{best}_i \leftarrow \mathbf{x}_i(t+1), \quad f^{p}_i \leftarrow f(\mathbf{x}_i(t+1))
```

```math
\mathbf{g}^{best}_g \leftarrow \arg\min_{i:\,g_i = g} f^{p}_i, \quad g=1,\dots,M
```

```math
\text{якщо } f^{p}_i < f^{best}: \quad f^{best} \leftarrow f^{p}_i
```

### Крок 7. Внутрішньогрупове імунне стиснення (кожні *z* ітерацій)

Активується якщо $t \mod z = 0$. Для кожної групи обчислити **радіус стиснення**:

```math
R_{in}(t) = \frac{\sqrt{r}}{N_m \cdot M \cdot 2 \cdot (1 + t)}
```

де $N_m = |G_g|$ — кількість агентів у групі. Якщо два агенти ближче ніж $R_{in}$, **гірший із них видаляється та генерується наново** в околиці лідера групи (агента з найкращим $f^p$) з нормальним розподілом радіуса $\sigma$:

```math
\sigma = R_{in} \cdot \left( \frac{p_{min}}{1 - p_{min}} \cdot (N_m - 1) \right)^{1/r}
```

```math
\mathbf{x}_{idx} \leftarrow \mathbf{x}_{leader} + \boldsymbol{\xi}, \quad \xi_j \sim U(-\sigma, \sigma)
```

де $p_{min} = 0.05$ — імовірність випадкового збігу. Цей механізм усуває дублювання у популяції та підтримує її різноманітність.

### Крок 8. Міжгрупове змагання (кожні *f* ітерацій)

Активується якщо $t \mod f = 0$ та $M > 1$. Впорядкувати групи за найкращим $f^p$ в кожній:

```math
\text{order} = \text{argsort}_g \min_{i \in G_g} f^{p}_i
```

```math
\text{winner} = \text{order}[0], \quad \text{loser} = \text{order}[-1]
```

Оновити лічильники поразок:

```math
\text{losses}_{winner} = 0, \quad \text{losses}_{loser} \mathrel{+}= 1
```

### Крок 9. Розформування групи та перерозподіл агентів

Якщо програш накопичив $C_{max}$ поразок і груп більше однієї, **група розформовується**, а її агенти перерозподіляються до найближчих за центроїдом груп, що залишилися:

```math
\text{якщо } \text{losses}_{loser} \geq C_{max} \text{ і } M > 1:
```

```math
\forall\, i \in G_{loser}: \quad g_i \leftarrow \arg\min_{k \in \text{keep}} \|\mathbf{x}_i - \bar{\mathbf{x}}_k\|
```

```math
M \leftarrow M - 1, \quad \text{losses}_g = 0, \;\; \forall g
```

де $\text{keep} = \{1,\dots,M\} \setminus \{\text{loser}\}$. Це призводить до **адаптивного зменшення кількості груп** з часом — рій консолідується в найкращій області.

---

### Критерій зупинки

Алгоритм завершує роботу за однією з умов:

| Умова | Опис |
|---|---|
| $f^{best} \leq \varepsilon$ | Досягнуто заданої точності ($\varepsilon = 10^{-10}$) |
| Стагнація $\geq 150$ ітерацій | Немає поліпшення $f^{best}$ |
| $t = \text{max\_iter}$ | Вичерпано бюджет ітерацій |

Алгоритм повертає кортеж $(f^{best}, \text{Iter}_{best})$, де $\text{Iter}_{best}$ — ітерація, на якій було знайдено $f^{best}$.

---

### Метрика якості параметрів

Для оцінки та порівняння наборів гіперпараметрів у Таблицях 2.2–2.4 використовується метрика Зіборова:

```math
F_{param} = |f^{best}| \cdot \text{Iter}_{best}
```

— чим менше значення $F_{param}$, тим краще: алгоритм має одночасно знайти **точне** рішення (малий $f^{best}$) та зробити це **швидко** (мала $\text{Iter}_{best}$). Усереднення по $N$ запусках на кожен набір гіперпараметрів забезпечує статистичну стабільність:

```math
\overline{F}_{param} = \frac{1}{N} \sum_{k=1}^{N} |f^{best}_k| \cdot \text{Iter}_{best,k}
```
